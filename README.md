# 2ГИС — Экран офиса

Слайдшоу для телевизоров/мониторов в офисе. Показывает фото и видео по кругу.
Все экраны синхронны — открывают один URL и показывают один и тот же слайд.

---

## Быстрый старт на компьютере (локально)

Тебе нужен [Node.js](https://nodejs.org) (версия 18+) и [Vercel CLI](https://vercel.com/docs/cli).

```bash
# 1. Установи Vercel CLI (один раз)
npm install -g vercel

# 2. Установи зависимости проекта
npm install

# 3. Положи свои фото/видео в папку public/media

# 4. Собери манифест (список файлов)
npm run build

# 5. Запусти локальный сервер
vercel dev
```

Открой в браузере: **http://localhost:3000**

На телевизоре в той же сети открой: **http://[IP-компьютера]:3000**
Узнать IP: `ipconfig` (Windows) / `ifconfig` (Mac/Linux) — строка вида `192.168.x.x`.

## Где лежат слайды

Все медиафайлы кладёшь в папку:

```
public/media/
```

**После добавления/удаления файлов** обязательно пересобери манифест:

```bash
npm run build
```

Без этого новые файлы не появятся (манифест — это список файлов для плеера).

Порядок слайдов — по имени файла. Используй числовую нумерацию:
```
public/media/
  01_welcome.jpg
  02_team.jpg
  03_promo.mp4
  04_news.jpg
```

## Деплой на Vercel (показ через интернет)

### Шаг 1 — залей на GitHub

Создай репозиторий на [github.com](https://github.com) и залей всю папку проекта.

```bash
git init
git add .
git commit -m "init"
git remote add origin https://github.com/ТВОЙлогин/ИМЯрепо.git
git push -u origin master
```

### Шаг 2 — подключи Vercel

1. Зайди на [vercel.com](https://vercel.com) → **Add New → Project**
2. Выбери свой репозиторий
3. Framework Preset: **Other** (настройки менять не надо — всё в `vercel.json`)
4. Нажми **Deploy**

Через ~минуту получишь адрес вида `https://имя.vercel.app`.
Открывай его на всех телевизорах, жми F11 (полный экран) — готово.

### Добавление файлов после деплоя

Положи новые фото/видео в `public/media`, затем:

```bash
npm run build        # обновить манифест
git add .
git commit -m "add media"
git push             # Vercel сам пересоберётся (~1 мин)
```

## Рекомендации по видео (чтобы не лагало на ТВ)

Слабые телевизоры плохо справляются с тяжёлыми видеофайлами.
Для плавного воспроизведения подготовь видео так:

| Параметр | Рекомендация |
|---|---|
| Кодек | **H.264, profile Baseline или Main** |
| Разрешение | не выше 1920×1080 |
| Битрейт | 4–8 Мбит/с |
| Контейнер | **MP4** |
| Аудио | AAC, 128–192 кбит/с |

Команда для перекодирования через [ffmpeg](https://ffmpeg.org):

```bash
ffmpeg -i входной.mp4 -c:v libx264 -profile:v baseline -level 3.1 \
       -b:v 5M -c:a aac -b:a 128k -movflags +faststart выходной.mp4
```

Флаг `-movflags +faststart` перемещает метаданные в начало файла — видео начинает играть сразу, не ожидая полной загрузки.

## Управление звуком

Видео по умолчанию играет без звука (политика браузеров).
Чтобы включить звук — нажми/коснись экрана (появится кнопка «Нажмите для звука»).

На телефонах громкость регулируется только кнопками устройства, не сайтом.

## Частые вопросы

**Добавил файлы, но они не появились на сайте**
→ Запусти `npm run build` и задеплой снова.

**Видео лагает на телевизоре**
→ Перекодируй в H.264 Baseline (команда ffmpeg выше). iPhone-видео (HEVC/H.265) ТВ тянет с трудом.

**Хочу чтобы слайды не совпадали по времени на разных экранах**
→ Это сделано специально — все экраны синхронны. Если синхронизация не нужна, скажи, уберём.

**Как поменять время показа одного слайда?**
→ В файле `api/playlist.js` строка `imageDuration: 10` — это секунды для фото. Для видео — реальная длина ролика.

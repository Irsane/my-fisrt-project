# 2ГИС — Экран офиса на Vercel

Серверная синхронизация экранов на бесплатном хостинге Vercel.
Телевизоры открывают один адрес `https://<твой-проект>.vercel.app` и показывают слайды синхронно.

> Важно про Vercel: это serverless. Постоянного `node server.js` и папки, в которую
> можно «на лету» перетаскивать файлы, там **нет** — диск только для чтения.
> Поэтому фотки добавляются одним из двух способов ниже.

---

## Как залить проект на Vercel

### Вариант A — через GitHub (рекомендую)
1. Создай репозиторий на GitHub и залей туда содержимое этой папки.
2. На vercel.com → **Add New → Project → Import** свой репозиторий.
3. Framework Preset: **Other**. Build настройки трогать не надо (всё в `vercel.json`).
4. **Deploy**. Через минуту получишь адрес `https://имя.vercel.app`.

### Вариант B — через CLI
```bash
npm i -g vercel
cd 2gis-vercel
vercel          # первый деплой (preview)
vercel --prod   # боевой адрес
```

На телевизорах открывай полученный адрес и разворачивай на весь экран (`F11`).

## Как добавлять фотки — два режима

### Режим 1. Репозиторий (без настройки, но нужен push)
- Клади файлы в папку **`public/media`**.
- `git commit` + `git push` → Vercel сам пересоберётся, и слайды обновятся.
- Минус: каждое изменение = пересборка (обычно меньше минуты).

### Режим 2. Vercel Blob (то самое «просто перетащил файл»)
Заливаешь фотки в облачное хранилище через дашборд — **без пересборки**.
1. В проекте на Vercel: **Storage → Create Database → Blob** → создай store и подключи к проекту.
   Vercel сам добавит переменную `BLOB_READ_WRITE_TOKEN`.
2. Сделай один **Redeploy**, чтобы переменная подхватилась.
3. Дальше заливай файлы в Blob с префиксом пути **`media/`**:
   - через дашборд: Storage → твой Blob → **Upload**, имя файла указывай как `media/01_foto.jpg`;
   - или из терминала: `vercel blob put public/media/01.jpg --pathname media/01.jpg`.
4. Готово — новые файлы появляются на экранах сами (без деплоя).

Пока в Blob пусто, показываются картинки из `public/media` (запасной вариант).
Как только в Blob появятся файлы с префиксом `media/` — берутся они.

Порядок слайдов — по именам, поэтому нумеруй: `01_...`, `02_...`
Поддержка: JPG · PNG · GIF · WEBP · AVIF · SVG · MP4 · WEBM · MOV · M4V

## Почему экраны синхронны
Каждый телевизор спрашивает у сервера время (`/api/now`) и по нему вычисляет текущий слайд.
Часы на самих телевизорах могут врать — картинка всё равно совпадёт.

## Пара нюансов
- Адрес `*.vercel.app` **публичный** — кто знает ссылку, тот видит экран.
  Если нужно закрыть, включи Password Protection в настройках проекта (платная фича Vercel)
  или добавь свой простой токен.
- Видео лучше держать небольшими: на бесплатном тарифе есть лимиты на трафик.
  Для тяжёлых видео разумнее режим Blob (отдаётся с CDN).

## Структура
```
public/
  index.html          экран-плеер
  media/              фотки для режима «репозиторий» (тут лежат 3 тестовые)
api/
  now.js              время сервера (синхронизация)
  playlist.js         список медиа (Blob или репозиторий)
  _media-manifest.js  генерируется при сборке (не редактировать руками)
scripts/
  build-manifest.mjs  сканирует public/media при сборке
vercel.json           конфиг сборки
package.json
```
